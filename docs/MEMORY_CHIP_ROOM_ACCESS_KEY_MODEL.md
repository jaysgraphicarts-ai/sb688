# SB688 — Memory Chip Room + Access Key Model

**Repository:** `jaysgraphicarts-ai/sb688`  
**System:** SB688 → SB689 Sovereign Stitch / Hive Mind  
**Owner:** JGA Enterprise / Jay's Graphic Arts  
**Visibility:** Public-safe ownership and access doctrine  
**Status:** Activated as public architecture protocol  

---

## 1. Activation statement

The SB688 Memory Chip model is defined as a customer-owned business-memory access model.

JGA holds the clipped operating room and system socket.

The customer, business, artist, or owner holds the access key to their own private Memory Chip.

The Memory Chip links to that user's own business operation and serves their private memory through the Clip Bricks they rent monthly or own outright.

---

## 2. Clean model

```text
JGA HOLDS THE ROOM
CUSTOMER HOLDS THE ACCESS KEY
MEMORY CHIP HOLDS THE CUSTOMER'S BUSINESS MEMORY
CLIP BRICKS SERVE THE OPERATION
AVA OPERATES ONLY AFTER THE HANDSHAKE
VERA CHECKS IMPORTANT ACTIONS
LEDGER RECORDS THE SESSION
```

Public-safe explanation:

```text
JGA provides the room, socket, protocol, and clipped system.
The business owner provides the access key.
The Memory Chip belongs to that business operation.
The rented or owned Clip Bricks serve that private operation.
```

---

## 3. What the room means

The room is the JGA-controlled operating environment.

It can mean:

- the SB688/SB689 architecture
- the Hive Mind socket
- the ClipBrick connection rack
- the AVA session environment
- the VERA gate
- the ledger protocol
- the demo-to-live pathway
- the private backend infrastructure when deployed

The room is not the customer's private business memory.

The room is the controlled space where authorized business memory can be served through approved bricks.

---

## 4. What the access key means

The access key belongs to the business, artist, client, or owner.

It can be represented by:

- a Business Micro Mini Card
- QR card
- NFC card
- digital pass
- client portal login
- private activation token
- account credential
- authenticated session

The access key should not expose private memory directly.

It starts the STITCH handshake that links the user to the correct Memory Chip.

---

## 5. What the Memory Chip means

The Memory Chip is the private business-memory container.

It links to one authorized business operation.

It can hold or reference:

- business identity
- owner preferences
- brand rules
- services
- pricing
- customer notes
- workflow history
- approval queue
- connected Clip Bricks
- AVA permissions
- ledger checkpoints

The Memory Chip is not public.

The Memory Chip belongs to the user's own business operation and should not be mixed with another business.

---

## 6. Renting or owning Clip Bricks

A user may access business functionality through Clip Bricks that are either:

1. **Rented monthly** — subscription/lease access.
2. **Owned outright** — paid-out or lifetime access, depending on agreement.

Examples:

- Sales Brick
- Brand Brick
- Customer Brick
- Content Brick
- Booking Brick
- Support Brick
- Finance Prep Brick
- Artist Ops Brick
- Merch Brick
- Print Fulfillment Brick
- Verification Brick

The brick serves the user's private operation only after the access key and Memory Chip handshake passes.

---

## 7. Handshake path

```text
USER PRESENTS ACCESS KEY
  ↓
STITCH HANDSHAKE STARTS
  ↓
BUSINESS ID MATCHED
  ↓
MEMORY CHIP FOUND
  ↓
LEASE / OWNERSHIP CHECK
  ↓
APPROVED CLIP BRICKS LOADED
  ↓
AVA SESSION CONNECTS
  ↓
VERA CHECKS RISK
  ↓
LEDGER RECORDS SESSION
```

If the handshake fails, the user stays in demo, denied, expired, read-only, or quarantine mode.

## 9. Privacy law

```text
One business Memory Chip may not read another business Memory Chip.
One access key may not unlock another owner's private operation.
A public demo may not access live business memory.
JGA public documentation may not include private Memory Chip contents.
```

Shared architecture does not mean shared private memory.

## 10. Public-safe card payload

A card or public access object should only hold routing-safe information.

Example:

```json
{
  "card_type": "JGA_BUSINESS_MICRO_MINI_CARD",
  "public_card_id": "PUBLIC_CARD_EXAMPLE",
  "system": "SB688/SB689",
  "handshake": "STITCH_REQUIRED",
  "memory": "PRIVATE_SERVER_SIDE"
}
```

Do not store private business data, API keys, customer records, or live Memory Chip contents on a public card.

## 11. Subscription and paid-out states

| State | Meaning |
|---|---|
| `DEMO` | No private memory access. Public preview only. |
| `LEASED_MONTHLY` | User rents access to approved Clip Bricks monthly. |
| `PAID_OUT` | User owns approved access under agreement. |
| `EXPIRED` | Lease expired. Live writes disabled. |
| `READ_ONLY` | Archive/export/renewal state. |
| `QUARANTINED` | Risk, corruption, invalid key, or security issue. |
