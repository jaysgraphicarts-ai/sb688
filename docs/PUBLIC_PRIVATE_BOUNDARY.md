# Public / Private Boundary

**Domain:** `jgaos2026-gif.com`  
**Repo:** `jaysgraphicarts-ai/sb688`  
**System:** SB688 / SB689

---

## Public repository role

The public SB688 repository explains the architecture, public doctrine, onboarding flow, and system language.

It is used for:

- public system explanation
- sales-safe documents
- onboarding templates
- public prompt session access
- roadmap
- claim boundaries
- architecture diagrams and descriptions

---

## Private system role

Private infrastructure is used for:

- live runtime code
- private business operations
- issued Memory Chips
- access pass secrets
- client records
- private ledgers
- service credentials
- deployment configuration
- live AVA sessions

---

## Rule

```text
Public explains the system.
Private operates the system.
```

---

## Do not publish publicly

- client or customer records
- live Memory Chip contents
- card or access secrets
- API keys
- passwords
- private ledgers
- production database credentials
- private runtime configs
- unredacted payment records

---

## Safe to publish publicly

- public architecture
- system names
- public onboarding forms
- public prompt doctrine
- high-level system maps
- non-sensitive roadmap
- claim boundary
- sales-safe descriptions

---

## Honest claim boundary

The public repository documents the SB688 architecture. It does not prove live production deployment by itself.

Live operation requires private hosting, authentication, database, monitoring, and authorized Memory Chip access.
